# 個人記帳簿系統 (Personal Expense Tracker) PRD

## 1. 專案概述

- **背景與動機**：為了更好地管理個人財務，追蹤每一筆收入與支出，需要一個工具來掌握資金流向，避免不必要的浪費，幫助達成儲蓄與理財目標。
- **目標用戶**：我（有個人財務管理需求的使用者）
- **核心價值主張**：提供簡單、直覺的記帳體驗，並透過清晰的總餘額統計與數據視覺化，幫助使用者快速且準確地掌握個人財務狀況。

---

## 2. 功能需求

以下為系統核心功能，並附上對應的使用者故事（User Story）：

1. **收支記錄管理**
   - 作為**個人使用者**，我希望**能夠快速新增、檢視、修改與刪除每一筆收入與支出記錄**，以便**隨時保持財務資料的準確性與最新狀態**。
2. **多樣化分類系統**
   - 作為**個人使用者**，我希望**能夠使用預設分類或為收支建立自訂分類（如：餐飲、交通、薪水）**，以便**清楚分辨每筆錢的來源與去向**。
3. **數據統計與視覺化報表**
   - 作為**個人使用者**，我希望**系統能顯示總餘額，並自動產生圓餅圖或長條圖等視覺化圖表**，以便**快速掌握每週、每月的財務狀況與收支趨勢**。
4. **預算提醒與超支警告**
   - 作為**個人使用者**，我希望**能夠針對特定類別設定每月預算，當支出接近或超過預算時顯示視覺化警告**，以便**及時控制花費，避免超支**。
5. **多帳戶/錢包追蹤**
   - 作為**個人使用者**，我希望**可以建立多個帳戶（如：現金、銀行帳戶、信用卡）並獨立記錄其收支與餘額**，以便**統一管理散落各處的資金**。

---

## 3. 非功能需求

- **技術限制**：
  - 後端框架：使用 Python + Flask
  - 前端渲染：使用 Jinja2 模板引擎與原生 HTML/CSS/JS (不依賴複雜前端框架)
  - 資料庫：使用 SQLite，便於本地部署與輕量化儲存
- **效能與安全考量**：
  - 系統操作及頁面載入時間應極小化，確保流暢的記帳體驗。
  - 需要基本的資料驗證防護（例如：金額不得為負數，防範 SQL Injection 與 XSS）。
  - 資料儲存於本地端資料庫，確保個人財務隱私安全。

---

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須有)**：
  - 核心的收支記錄管理（CRUD：建立、讀取、更新、刪除）。
  - 基本的預設收支分類系統。
  - 首頁總餘額與單月收支總計顯示。
- **Should Have (應該有)**：
  - 簡單的月度數據報表（如圖表呈現收支比例）。
  - 多帳戶/錢包的建立與追蹤。
- **Nice to Have (可以有)**：
  - 預算提醒與超支警告提示機制。
  - 自訂新增分類與標籤功能。
  - 收支記錄 CSV 匯出功能。
